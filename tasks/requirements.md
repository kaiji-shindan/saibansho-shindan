## 1. ビジョン・ミッション

### ビジョン
SNS上の誹謗中傷を客観的に可視化し、被害者の救済と発信者の自覚を促すインフラとなる。

### ミッション
- **被害者**: 自分が受けている誹謗中傷を、感覚ではなく**客観的データ**で把握できる状態にする。
- **発信者**: 自分の投稿が法的リスクを孕んでいないか、自然に**セルフチェック**できるようにする。
- **社会**: 誹謗中傷が「特定可能で、対処可能なもの」であるという認知を広げる。

### 事業上のゴール
**提携弁護士事務所への B2B 商品**として、本ツールを売却またはライセンス提供する。
ユーザーからの直接課金は行わない（弁護士法72条の非弁行為回避）。

## 2. ローンチ戦略

### 重要な前提
**提携弁護士事務所はリリース時点で確保していない**。アプリ自体を実績・営業ツールとして使い、リリース後に弁護士事務所へ営業をかける。

### 段階的ロードマップ
| Phase | 期間 | 主な内容 |
|---|---|---|
| Phase 1 | リリース前 | 弁護士関連文言の中立化、法務文書整備、1ページ完結UI、基本的な分析機能 |
| Phase 2 | リリース後 1〜2ヶ月 | LINE登録 + X OAuth による深堀り分析、加害者プロファイリング、PDFレポート |

| Phase 4 | 提携獲得後 | 「提携弁護士事務所への取次」文言を復活、管理画面のB2B本格運用 |

## 4. 機能要件

### 4.1 Layer 1: 無料診断（リリース必須）

#### 入口（`/`）
- X username 入力フォームのみ（最小UI）
- ソーシャルプルーフ（累計分析数、診断回数）
- 法的助言ではない旨の注記
- 法務文書（利用規約・プライバシーポリシー・特商法）へのリンク

#### 診断結果（`/diagnose/[username]`）— **1ページ完結**
**重要**: タブ・モード切替・ステップUI は使用しない。全情報を縦スクロール1ページで提示する。

構成順序:
1. アカウントプロフィールサマリ
2. 総合スコア（法的リスクレベル S〜E、0-100点）
3. カテゴリ別件数（名誉毀損・侮辱・脅迫・プライバシー侵害）
4. 該当投稿リスト（上位3件）
5. 投稿行動データ（リプライ率・問題投稿率・月別推移・メンション先・該当語）
6. 詳細レポート CTA（LINE登録 → Layer 2）
7. 想定される対応の選択肢（一般情報、法的助言ではない）
8. 開示請求の費用相場（参考情報、出典明示）
9. 開示請求書テンプレートへのリンク
10. 注意書き・免責事項

#### 入力データ
- X API v2 Basic tier から公開ツイート最大100件取得
- Anthropic Claude による分類（名誉毀損／侮辱／脅迫／プライバシー侵害／該当なし）
- 独自スコアリング（severity weight × normalized）

#### 出力データ
- DiagnosisData 型（`src/lib/diagnose-types.ts`）
- TTL 24時間で Supabase キャッシュ

### 4.2 Layer 2: 深堀り分析（Phase 2、リリース後実装）

#### LINE 登録 → X OAuth 連結フロー
**シームレス**を最優先。独立した同意フォーム・チェックボックス付きモーダルは作らない。

```
[診断結果ページの CTA]
  → LINE 友だち追加 (lin.ee)
  → 自動応答メッセージ内のリンクをタップ
  → 専用ランディング（短文）「Xを連結すると過去全件分析できます」
  → 「X連結」ボタン直下に「連結により プライバシーポリシー に同意」の小注記
  → X OAuth 認可画面（X側の標準フロー）
  → 詳細レポートに直接リダイレクト
  → LINE で完了プッシュ通知
```

#### 機能
1. 自分宛メンション・リプライ全件分析（過去最大3,200件）
2. 時系列の被害推移グラフ
3. 加害アカウント集計（頻出送信元）
4. 削除前のスナップショット保存（運営DB）
5. 開示請求書テンプレートへのデータ自動流し込み
6. PDFレポート出力（弁護士に共有可能な形式）

#### 技術上の注意
- **DM 取得は実装しない**: X API v2 では他人の DM にアクセス不可。Pro tier ($5,000/月) でも自分の DM 限定。当面はメンション分析に集中する。

### 4.3 Layer 3: 管理画面（B2B商品）

#### 弁護士事務所向けダッシュボード（`/admin`）
- Basic 認証（Phase 4 で本格的なIDP移行検討）
- KPI ダッシュボード（日次診断数、LINE登録数、流入元、UTM別CV）
- リード一覧（X username、LINE 連結状況、被害状況サマリ、UTM）
- リード詳細ビュー（被害者プロファイル、分析結果、加害者リスト、時系列、PDFダウンロード）
- CSV エクスポート
- 案件ステータス管理（未対応／連絡済／受任／クローズ）

#### `/for-lawyers` LP（営業 LP）
- 提供価値の説明（業務支援ツールとしての位置付け）
- KPI 数値の表示（運営側のダッシュボードからの抜粋）
- 管理画面プレビュー
- 問い合わせフォーム（メールリンクで可）

## 5. 非機能要件

### 5.1 パフォーマンス
- 診断 API レスポンス: P95 で 5 秒以内
- 1ページ完結のため LCP < 2.5s
- X API キャッシュ TTL 24時間

### 5.2 可用性
- Vercel Pro での運用
- Supabase 無料 tier から開始、流入次第で有料 tier 移行

### 5.3 セキュリティ
- 全通信 HTTPS
- `vercel.json` のセキュリティヘッダ（`X-Frame-Options`, `Referrer-Policy` 等）
- `/admin` `/api` は noindex
- `ADMIN_PASSWORD` は本番で必ず強パスワード化
- API レート制限（IP単位 10/分・60/時）

### 5.4 X API クォータ管理
- Basic tier $200/月 を前提
- レート制限超過時 429 を返す
- 分析結果は 24h キャッシュで API 呼び出し削減

### 5.5 個人情報保護
- 利用目的・保管期間・第三者提供は `/privacy` で明示
- LINE / X OAuth 連結時は通常Webサービス相当のフロー（独立した同意フォームなし）
- LINE ID・X ID と紐付けたユーザーデータは原則 Supabase で暗号化保管

## 6. データ設計

### 6.1 蓄積データ
| データ | 取得元 | 取得タイミング | 用途 |
|---|---|---|---|
| 診断対象 X username | フォーム入力 | Layer 1 | 基本識別子 |
| 診断結果（DiagnosisData） | X API + Claude | Layer 1 実行時 | キャッシュ・統計 |
| LINE user ID | LINE Webhook（将来） | LINE友だち追加時 | プッシュ通知ルート |
| X user ID（連結後） | X OAuth | Layer 2 連結時 | LINE×X 永続紐付け |
| アクセスログ・UTM | 全ページ | アクセス時 | 流入分析 |
| メンション分析結果 | Layer 2 実行 | Layer 2 実行時 | 証拠資産 |
| 加害者側 X user ID | Layer 2 結果集計 | Layer 2 完了時 | 二次資産（横断統計） |

### 6.2 主要テーブル（Supabase）
- `diagnose_cache`: 診断結果キャッシュ（TTL 24h）
- `leads`: リード記録（kind / UTM / セッション / 外部アカウント ID）
- `leads_by_campaign`: キャンペーン別集計 view

詳細は `db/schema.sql` 参照。

### 6.3 同意取得方針
- 独立した同意フォーム・チェックボックス付きモーダルを**作らない**
- プライバシーポリシーで利用目的・第三者提供を明示
- LINE / X OAuth ボタン直下に小サイズの「同意したものとみなします」注記リンク

## 7. UI/UX 方針

### 7.1 トーン
- **真面目・社会的意義・業務ツール感**で統一
- 弁護士事務所が見て「業務に使える」と判断できる粒度
- 公益性のある情報提示として表現

### 7.2 禁止事項（提案・実装ともに）
- ❌ ランキング・偏差値・「上位5%」「珍しいタイプ」等の統計コメント
- ❌ ゲーミフィケーション要素・扇情的なキャッチコピー
- ❌ セルフチェック / 被害確認 のモード切替UI（1ページ完結）
- ❌ 独立した同意フォームページ・チェックボックス付き同意モーダル
- ❌ ユーザーへの直接課金・サブスクリプションプラン
- ❌ 弁護士不在期間中の「提携弁護士事務所への取次」「無料相談」「弁護士紹介」文言

### 7.3 推奨表現
- ❌ 「違法投稿が◯件」 → ⭕ 「法的リスクの可能性がある投稿が◯件」
- ❌ 「弁護士に相談すべき」 → ⭕ 「専門家への相談を検討する材料」
- ❌ 「あなたの被害リスク偏差値」 → ⭕ 「過去30日の該当投稿: ◯件」

### 7.4 シェア機能
- OG画像でスコアカードを共有可能
- シェア文言は事実ベース・中立（「私のアカウントへの誹謗中傷を可視化しました」程度）

## 8. 法務制約

### 8.1 弁護士法72条（非弁行為）回避
- 法的判断・法的助言・代理行為は一切提供しない
- 全ての結果に「法的助言ではない」旨を明記
- 個別事案への対応提案を行わない

### 8.2 個人情報保護法対応
- プライバシーポリシーで利用目的を明示
- 第三者提供（将来の提携弁護士事務所への取次）を明記し、ユーザーに予見可能にする
- 保管期間・問い合わせ窓口を提示

### 8.3 X 開発者規約遵守
- 公開データ以外への不正アクセスをしない
- DM の不正取得をしない（技術的にも不可能）
- レート制限を守る

### 8.4 リリース前の法務監査（強く推奨）
- 顧問弁護士へのスポット相談 1 回（5〜10万円）
- 監査対象: プライバシーポリシー / 利用規約 / 診断結果の文言 / 全体的な非弁該当性
